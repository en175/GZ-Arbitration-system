## **一、总体思路**

### **1.1 总体思路**

> **围绕立案与办案两大核心作业场景，重构智能化作业中枢，并向多端协同、角色工作台、管理驾驶舱延展。**

### **1.2 规划结构**

**两个核心作业中枢 + 三类延展能力**

| 层级 | 模块 | 定位 |
|----|----|----|
| 核心作业中枢 | 立案智能审核工作台 | 面向立案秘书，重构立案审核过程中的材料理解、问题识别、字段核对与处理链路 |
| 核心作业中枢 | 案件总作业台 | 面向办案秘书、仲裁员、调解员，重构案件认知、阅卷、研判、协同沉淀链路 |
| 延展能力 | 角色工作台 | 面向不同角色提供待办聚合、重点提醒、快捷入口和工作视图 |
| 延展能力 | 多端协同 | 支撑内部审批跨端处理与对外协同能力升级 |
| 延展能力 | 管理驾驶舱 | 面向管理人员提供案件推进、负荷、效率、风险等可视化能力 |

## **二、核心模块**

## **2.1 立案智能审核**工作台

### **2.1.1 模块定位**

> **面向立案阶段的受理前置判断中枢。**

### **2.1.2 与案件总作业台边界**

| 维度 | 立案智能审核工作台 | 案件总作业台 |
|----|----|----|
| 所处阶段 | 立案前 / 受理审核阶段 | 立案后 / 办案处理阶段 |
| 核心目标 | 判断是否齐、是否准、是否满足受理与补正处理要求 | 判断怎么看、怎么跟、怎么继续处理案件 |
| 核心用户 | 立案秘书为主，兼顾办案秘书、仲裁员后续承接 | 办案秘书、仲裁员为主，兼顾立案阶段回看 |
| 核心对象 | 申请材料、表单字段、受理规则、问题项、补正项 | 全案卷宗、案件脉络、争议焦点、当前待办、协同记录 |
| 核心产物 | 审核结论、问题清单、补正要点、字段确认结果 | 案情摘要、时间线、争议焦点、研判记录、协同动态 |
| 不重点承载 | 深度案情研判、争议焦点展开、协同沉淀 | 受理校验、补正生成、字段入库前确认 |

### **2.1.4 页面形态**

> **左侧材料区 + 中间核对区 + 右侧问题与处理区**

**左侧：材料区**

**中间：核对区**

## **2.2 案件总作业台**

### **2.2.1 模块定位**

> **先看懂案件、再继续处理案件。把卷宗材料、案件脉络、当前待办、协同留痕与 AI 辅助能力组织到同一作业空间中，形成案件办理阶段的核心工作台。**

### **2.2.2 能力结构**

1.  **首屏认知层**

解决：用户刚进入案件时先看什么，目标是帮助用户用最短时间建立基础认知。

包含：

- 案件基础信息卡

- 当前阶段 / 当前节点 / 当前状态

- AI 首屏案情摘要

- 当前关键提醒

- 最近关键动态

2.  **卷宗理解层**

解决：案件材料怎么看得更快、更清楚，是本模块最关键的 AI 价值承载层之一。

包含：

- 卷宗材料目录化与结构化组织

- 案件时间线

- 争议焦点归纳

- 材料与时间线 / 争议焦点的关联跳转

- 关键材料快捷预览与定位

3.  **当前处理层**

解决：当前案件具体要怎么往下办。

包含：

- 当前待办展示

- 系统流程待办与人员工作待办双视图

- 当前节点需关注的信息提示

- 协同备注、提醒、轻量推进动作

- 最近处理记录

4.  **沉淀复用层**

解决：这个案件之前是怎么理解、怎么处理、怎么协同过的。

包含：

- 研判记录

- 人工确认过的关键结论沉淀

- 协同动态

- 处理留痕

- 可供后续接续的案件过程记录

### **2.2.4 页面形态**

**左侧案件导航 / 中间主工作区 / 右侧智能与待办侧栏**

| 区域 | 主要内容 |
|----|----|
| 左侧导航 | 案件基础导航；卷宗材料目录树；总览 / 时间线 / 争议焦点 / 研判记录 / 协同动态等视图入口 |
| 中间主工作区 | 首屏案件总览、时间线主视图、争议焦点主视图、材料关联主视图、研判记录查看与编辑、当前节点上下文信息 |
| 右侧侧栏 | 当前待办、重点提醒、AI 生成的当前要点提示、风险提示、快捷协同动作 |

## **三、延展能力**

## **3.1 角色**工作台

### **3.1.1 模块定位**

> **面向不同角色的日常工作入口与工作视图，围绕"待办聚合、案件聚合、重点提醒、快捷进入"四类核心诉求，形成从工作入口到具体案件处理的高频使用闭环。**

### **3.1.2 覆盖角色**

| 角色 | 工作台重点围绕 |
|----|----|
| 立案秘书 | 待审案件快速聚合、材料异常与缺件提示、补正相关事项提醒、字段确认与处理入口前置 |
| 办案秘书 | 当前待推进案件聚合、临期、超期、待确认事项前置、当前协同事项与流程提醒集中展示、快速进入案件总作业台继续处理 |
| 仲裁员 | 待查看、待确认、待处理事项聚合； <br>重点案件与关键材料前置待确认研判、待处理事项、关键提醒集中展示； <br>直接进入案件总作业台查看摘要、焦点、材料和记录； |
| 调解员 | 待查看、待确认、待处理事项聚合； <br>重点案件与关键材料前置； <br>待确认研判、待处理事项、关键提醒集中展示； <br>直接进入案件总作业台查看摘要、焦点、材料和记录； |

### **3.1.3 核心能力**

| 能力方向 | 聚合内容 |
|----|----|
| 待办聚合 | 流程待办、工作待办、待确认事项、待补正事项、待查看 / 待处理事项 |
| 我的案件 | 我负责的案件、我参与的案件、我待处理的案件、近期重点案件 |
| 重点提醒 | 临期事项、超期事项、缺件事项、待补正事项、待确认事项 |
| 快捷入口 | 立案智能审核工作台、案件总作业台、研判记录入口、重点材料入口、高频处理入口 |
| 个人视角筛选 | 优先级、状态、案件阶段、处理时效、是否异常、是否待确认 |

### **3.1.4 角色差异化**

| 角色 | 首屏重点建议 |
|----|----|
| 立案秘书 | 待审案件、材料异常、缺件 / 待补正、待确认字段、快速进入立案审核工作台 |
| 办案秘书 | 当前推进案件、协同待办、临期案件、节点提醒、快速进入案件总作业台 |
| 仲裁员 | 待查看案件、关键材料、争议焦点相关入口、待确认研判、待处理事项 |
| 调解员 | 待处理案件、案件摘要、轻量记录入口、关键提醒 |

| 功能点 | AI具体做什么 | 主要输入 | 技术方式 | 输出结果 | 人工确认 |
|----|----|----|----|----|----|
| 我的待办聚合 | 把不同来源待办统一归并，减少用户自己翻多个列表 | 流程待办、协同提醒、审批事项、异常事项 | 规则聚合 + 类型归类 | 待办聚合列表 | 无需确认 |

| 条件筛选辅助 | 根据用户常用操作、当前工作状态推荐筛选组合 | 用户操作记录、案件状态、待办分布 | 统计规则 + 轻量推荐 | 默认筛选项 / 常用筛选 | 无需确认 |
| 近期工作状态反馈 | 自动汇总"当前待办压力、近期处理量、临期事项数" | 待办数据、处理记录、状态分布 | 规则统计 + LLM摘要 | 轻量效率反馈卡 | 仅展示 |

### **3.1.6 页面形态**

> **上方重点提醒 + 中部工作聚合区 + 侧边快捷入口 / 筛选区**

**上方：重点提醒区**

用于展示当前最需要优先关注的事项，例如：

- 临期案件

- 超期案件

- 缺件案件

- 待补正事项

- 待确认事项

这一层应尽量短、直观、可点击进入，不宜写成长列表。

**中部：工作聚合区**

- 我的待办

- 我的案件

其中：

- 我的待办更偏动作入口

- 我的案件更偏案件回流入口

两者可以并列，也可以通过 tab 切换。

**侧边：快捷入口 / 个性筛选区**

- 立案审核工作台入口

- 案件总作业台入口

- 高频功能入口

- 筛选条件

- 轻量效率反馈

## **3.2 多端协同**

### **3.2.1 模块定位**

> **让关键案件信息、关键待办、关键处理动作能够在电脑、平板、手机之间连续衔接，使用户在不同设备上都能接住当前工作，而不是只能回到 PC 端才能继续。**

### **3.2.2 设备分工**

1.  **电脑端**

- 立案智能审核工作台的完整处理

- 案件总作业台的完整使用

- 材料深度浏览与核验

- 研判记录编辑

- 多信息并行查看

- 复杂处理流程操作

2.  **平板端**

- 案件摘要与总览查看

- 时间线、争议焦点查看

- 关键材料阅读

- 待办与提醒查看

- 轻量审批 / 确认 / 备注

- 最近动态与处理记录查看

3.  **手机端**

- 临期 / 超期 / 待确认提醒触达

- 快速查看当前待办

- 快速查看案件摘要

- 快速看关键材料

- 完成简单审批与确认动作

- 补充简短备注

- 查看处理结果和最新动态

### **3.2.3 典型使用场景**

**场景一：办案秘书离开工位时的待办处理**

办案秘书在办公室主要使用电脑处理案件，但在会议、沟通或短时间离开工位时，仍可能收到新的待确认事项、临期提醒或协同通知。

- 查看提醒内容

- 进入待办详情

- 查看案件摘要

- 查看关键材料

- 做出确认 / 退回 / 简要备注等动作

**场景二：仲裁员非 PC 场景下的快速查看与确认**

- 快速看某个案件的摘要

- 看关键材料和焦点

- 看最新处理记录

- 完成某些关键确认或审批动作

**场景三：平板作为移动阅读与阅卷辅助设备**

- 阅读体验优化

- 关键信息聚合查看

- 轻量确认动作承接

**场景四：电脑端与移动端之间的连续流转**

例如用户在电脑端处理案件处理到一半，离开工位后在手机上收到某个节点提醒，可以先在手机端看摘要、看待办、做简要确认，回到电脑后继续完整处理。

### **3.2.5 页面形态**

1.  **电脑端**

- 信息全

- 操作链完整

- 支持并行查看与深度处理

2.  **平板端**

> **阅读优先 + 轻处理补充**

重点是：

- 关键信息聚合

- 阅读舒适

- 轻量动作顺手

- 避免过度复杂表单

3.  **手机端**

> **提醒优先 + 待办优先 + 摘要优先**

重点是：

- 打开即看到关键事项

- 减少层级

- 减少输入

- 单次只承接短链路动作

## **3.3** **管理驾驶舱**

### **3.3.1 模块定位**

> **面向管理角色的案件推进总览与异常识别视图，围绕"看整体、看风险、看效率、看分布、看问题"五类核心管理诉求，形成对案件运行状态和组织处理状态的统一观察窗口。**

### **3.3.2 重点内容**

**一、案件推进总览**

- 案件总量

- 不同阶段案件分布

- 处理中案件情况

- 临期 / 超期案件情况

- 当前重点关注案件数量

**二、节点卡点与异常分布**

- 哪些流程节点更容易积压

- 哪些节点临期、超期更集中

- 哪些案件长时间停留在某一环节

- 哪些异常问题反复出现

- 哪些问题具有共性，而不是个案偶发

**三、案件风险与重点关注**

- 临期案件

- 超期案件

- 多次退回或反复流转案件

- 长时间无推进案件

- 存在明显异常或待协调事项的案件

**四、人员与承载情况**

- 不同人员当前承接案件情况

- 待办压力分布

- 临期 / 超期事项在人员维度上的分布

- 当前承载较重或推进压力较大的人员情况

**五、管理提醒与问题归纳**

- 当前最值得关注的异常类型

- 当前最集中的卡点环节

- 当前最需要协调推进的案件类型或事项类型

- 近期异常变化趋势
