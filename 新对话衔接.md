### EditorArea.vue 关键 Props 与 Expose

**Props**（`src/components/editor/EditorArea.vue`）：
- `modelValue`: String — 初始 HTML 内容
- `sections`: Array — 左侧章节导航列表 `[{ key, label }]`
- `editable`: Boolean (default: true) — false 时只读，隐藏工具栏
- `checkParagraphIds`: Array — 需高亮的校对段落 ID 列表
- `commentMarkers`: Array — 批注标记位置列表
- `placeholder`: String

**Expose**：`scrollToParagraph`, `scrollToCheckParagraph`, `scrollToCommentParagraph`, `highlightParagraph`, `insertContentAndHighlight`, `replaceSelectionText`, `getHTML`, `setContent`

智能阅卷视图中，用 `:editable="false"` + `:deep(.editor-toolbar) { display: none }` 隐藏工具栏。

### 文书写作功能特性

- 材料引用：高风险冲突（`conflictLevel: 'high'`）的证据条目点击引用时会弹出拦截提示
- AI 写作助手：按章节（`DOC_SECTIONS`）+ 预设指令（`SECTION_AI_PRESETS`）两级选择，回车或点击发送，返回 `AI_PRESET_RESPONSES` 中的 Mock 回复，可一键插入编辑器
- 校对检查：点击条目定位到对应 `paragraphId` 段落，通过 `editorRef.scrollToCheckParagraph([id])` 实现
- 人工批注：在编辑器选中文本后右键或点击添加批注按钮，批注绑定选中文本的 paragraphId；支持标记"已处理"
- Word 导入：mammoth 转 HTML 后 `setContent` 注入编辑器
- Word 导出：docx 库将当前 HTML 解析生成 .docx 下载

---

### 智能阅卷功能特性

- 左侧模块选择器（核心诉求/无争议事实/争议焦点）点击后：
  1. 编辑器滚动到对应 `paragraphId`（通过 `REVIEW_MODULES` 配置）
  2. 右侧审查建议切换为对应模块的 `MODULE_INSIGHTS`
- 材料列表点击"预览"打开弹窗，显示 `previewText`
- 模块选择器还有详情弹窗（coreClaim 显示 bullet list，undisputedFact 显示 sections，justiceFocus 显示 focuses 对抗结构）
- 右侧质量评分 5 项：logic/focus/discretion/fact/standard，分数 ≥90 显示绿色，否则橙色

### 开发约定与约束

1. **不使用 TypeScript**：所有组件、配置文件均为 `.js` / `.vue`（JS）
2. **不引入新 UI 框架**：仅使用 Element Plus 或自定义 CSS，不引入 Ant Design / Naive UI 等
3. **不使用 SCSS/Tailwind**：所有样式用 CSS 自定义属性 + scoped CSS
4. **Mock 数据与案件一致**：任何新增的演示数据都应使用粤仲案字第5412号案件的真实内容
5. **组件迁移原则**：从 TypeScript/Ant Design 参考项目迁移时，去除类型标注，将 `defineProps<{}>()` 改为 `defineProps({})`
6. **`:deep()` 用于跨组件样式**：在父组件中用 `:deep(.子组件类名)` 覆盖子组件样式，不破坏 scoped
7. **编辑器背景**：`.editor-container { background: #fff }`（白色，非灰色）
8. **不自动提交代码**：除非用户明确要求，否则不执行 git commit/push
